# Planning: Bank Statement Reconciliation & Tally Design

This document details the CSV/Excel statement parser, match/mismatch heuristic algorithms, and reconciliation interface to auto-tally ledger logs.

---

## 🎯 Target Reconciliation Features

1. **Statement Upload:**
   - Support standard CSV and Excel templates generated by major Indian banks (SBI, HDFC, ICICI, Axis).
2. **Auto-matching Heuristics:**
   - Automatically match uploaded bank rows against manual entries inside SQLite using:
     - Exact Transaction Amount.
     - Date variance (e.g., date of transfer is +/- 3 days of the bank value).
     - UPI Ref No / Transaction ID lookup.
3. **Difference Flagging & Ledger Tallying:**
   - Flag "Difference Entries":
     - Bank rows not matched in our ledger (e.g., hidden bank charges, interest credits, forgotten payments).
     - Ledger entries not cleared by the bank (e.g., checks issued but not yet cleared).
   - Easy one-click creation of ledger entries from unrecognized bank rows.

## 💾 Database Schema

```sql
-- Track state of uploaded bank statements
CREATE TABLE IF NOT EXISTS bank_statements (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  account_id INTEGER NOT NULL,
  uploaded_at TEXT DEFAULT (datetime('now')),
  filename TEXT NOT NULL,
  statement_start_date TEXT,
  statement_end_date TEXT,
  FOREIGN KEY (account_id) REFERENCES accounts(id)
);

-- Temp/raw rows imported from bank statement for parsing
CREATE TABLE IF NOT EXISTS statement_rows (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  statement_id INTEGER NOT NULL,
  date TEXT NOT NULL,
  description TEXT,
  ref_no TEXT,
  withdrawal_amount REAL DEFAULT 0,
  deposit_amount REAL DEFAULT 0,
  balance REAL NOT NULL,
  is_reconciled INTEGER DEFAULT 0,
  matched_transaction_id INTEGER,
  FOREIGN KEY (statement_id) REFERENCES bank_statements(id),
  FOREIGN KEY (matched_transaction_id) REFERENCES transactions(id)
);
```

## ⚙️ Reconciliation Match Algorithm

```javascript
// lib/reconciler.js
import db from './db';

/**
 * Scan unmatched statement rows and compare against unmatched ledger entries
 */
async function autoMatchStatement(statementId, accountId) {
  const statementRows = await db.queryAll(
    "SELECT * FROM statement_rows WHERE statement_id = ? AND is_reconciled = 0", 
    [statementId]
  );

  for (const row of statementRows) {
    const amount = row.withdrawal_amount > 0 ? row.withdrawal_amount : row.deposit_amount;
    const direction = row.withdrawal_amount > 0 ? 'debit' : 'credit';
    
    // Search ledger transaction matching same account, amount, direction, and near date
    const matchedTx = await db.queryOne(
      `SELECT id FROM transactions 
       WHERE account_id = ? 
         AND amount = ? 
         AND direction = ? 
         AND is_reconciled = 0 
         AND abs(julianday(date) - julianday(?)) <= 3 
       LIMIT 1`,
      [accountId, amount, direction, row.date]
    );

    if (matchedTx) {
      // Auto-reconcile both
      await db.run(
        "UPDATE statement_rows SET is_reconciled = 1, matched_transaction_id = ? WHERE id = ?",
        [matchedTx.id, row.id]
      );
      await db.run(
        "UPDATE transactions SET is_reconciled = 1 WHERE id = ?",
        [matchedTx.id]
      );
    }
  }
}
```

## 🖥️ UI Dashboard Structure (`app/reconciliation/page.js`)

*   **Tally Interface (Split Screen):**
    *   **Left Column (Bank Statement Rows):** Unmatched bank records with deposit/withdrawal indicators.
    *   **Right Column (Our Ledger Entries):** Manual logs waiting to be cleared.
*   **Reconciliation Matcher Card:** Displays dynamic matching suggestions (e.g. *"We found a match for ₹1,200 UPI payment on 12-June. Clear this entry? [Yes] [No]"*).
*   **Quick Add Action:** Select an unmatched bank row (e.g., standard SMS alert interest credit of ₹42) and click a button to immediately add it as a new transaction in the ledger.
